# RAIF Standard

The repairable, schema-first structured-output protocol for probabilistic generators (LLMs). Separates model-friendly emission syntax from strict canonical transport syntax, so an interpreter can repair, validate, canonicalize, and convert model output into JSON or typed tool-call arguments.

## Language

**RAIF-G** (Generation Format):
The loose, low-entropy variant emitted directly by the model. No checksums, no length prefixes; optimized for model fluency and grammar-constrained decoding.
_Avoid_: "RAIF input", "model RAIF"

**RAIF-R** (Repairable Canonical Format):
The strict variant produced by the interpreter from RAIF-G. Length-prefixed fields, field and object checksums, canonical field ordering. Used for transport, audit, replay.

**Position mode** (`m=pos`):
Schema-driven emission. Fields are addressed by integer position (`1:`, `2:`); the schema maps positions to field names and types. Lowest token count; eliminates field-name hallucination.

**Named mode** (`m=named`):
Schema-light emission. Fields are addressed by name (`subject:s=...`). Verbose, human-readable, used for debugging.

**Path mode** (`m=path`):
Arbitrary-JSON emission. Each leaf is addressed by a JSON path (`user.name:s=...`, `tags[0]:s=...`). The bridge that makes JSON ↔ RAIF round-trip possible.

**Text block**:
A delimited raw value. The cheap forms are `<<<value>>>` (single line — embedded `>>>` is fine; decoder unwraps by outermost slice) and `<<<\n...\n>>>` (bare multiline — used when value contains `\n`/`\r` and no line literally equals `>>>`). The robust form `<<<NONCE\n...\n>>>NONCE` is reserved for the rare multiline value where a content line is literally `>>>`. See ADR-0011.

**Nonce**:
A short random hex string generated by the encoder per text block and used as the suffix on the opener and closer (`<<<7f2a` / `>>>7f2a`). **Optional**: only emitted when a content line of a multiline value literally equals `>>>`. Bare-line multiline (`<<<\n...\n>>>`) is the default. See ADR-0011.

**Inline-object leaf** (`prefix={k1=v1,k2=v2}` / `arr[N]={k=v,k=v}`):
A single-leaf representation of a non-empty object with all-primitive cell values. Used when shorter than path expansion. Targets heterogeneous arrays (where table mode is ineligible) and flat nested objects with long parent prefixes. Keys and values inside follow the standard wrap rules with `,` `{` `}` added as wrap triggers. Nested objects or arrays inside an inline-object are not allowed; the encoder falls back to path mode for non-flat structures. See ADR-0010.

**Cheapest-mode pick**:
For each array and each non-empty nested object the encoder builds every legal candidate (path mode, table mode, inline-object mode, array-literal mode) and emits the shortest by byte length. Ties resolve by preference order: path > table > inline > literal. The decoder accepts all forms unchanged. See ADR-0012.

**Array literal** (`prefix=[\n…rows…\n]`):
A single-leaf-block representation of a non-empty array. The opener (`prefix=[`) shares the array prefix once; each subsequent line is one element (a primitive or a flat inline-object); a line equal to `]` closes the block. Beats path mode and inline-object mode whenever the array has 2+ rows and the parent prefix is non-trivial. The encoder additionally wraps any string equal to `[` (avoid opener collision) and any row equal to `]` inside a literal (avoid early close). See ADR-0013.

**Repair**:
A bounded, deterministic, auditable transformation applied by the interpreter to RAIF-G that fixes **syntax** — wrappers, delimiters, separators, field order, missing newlines, recoverable length/checksum errors. Repair never mutates field values.

**Validation**:
The check that field values conform to their declared type and (where applicable) schema constraints. A failed validation surfaces an error to the caller; the interpreter does not attempt to "fix" a bad value. Re-prompting the model is the standard recovery path.

**Leaf**:
A single RAIF-G line representing one scalar value, one null, or one empty-container marker. The unit of recoverability: a malformed leaf damages only itself, never neighbors. In path mode, every JSON value (including each array element) is one leaf.

**Table mode**:
The wire form for an array whose elements are all flat objects with the same key set. A `prefix::col1,col2,col3` header declares the column list once; subsequent `prefix[N]=v1,v2,v3` rows carry only the values. Encoder auto-detects eligibility; ineligible arrays fall back to path mode. See ADR-0008.

**Table header**:
A leaf of the form `prefix::col1,col2,col3` that declares the column list for the table-mode rows that follow under the same `prefix`. The `::` (double-colon) distinguishes the header from a typed leaf or a bare value.

**Table row**:
A leaf of the form `prefix[N]=v1,v2,v3` whose cell count must match the column count of the table header declared for `prefix`. Cells are comma-separated; cells containing `,`, `>>>`, or matching a JSON literal are wrapped with `<<<...>>>`.

**Mode marker** (`<raif>` / `</raif>`, future `<|raif_start|>` / `<|raif_end|>`):
A pair of tokens the runtime uses to switch a fine-tuned model into RAIF emission discipline and back out. Analogous to Gemma 4's `<|tool_call|>` / `<tool_call|>` tokens. **Not** wire-format ceremony for individual objects; not counted in per-object token math. For the prompt-only MVP they are textual tags; for fine-tuned integration they become special tokens.
_Avoid_: "envelope", "wrapper" — those framings suggest per-object overhead, which mode markers are not.

**Canonical RAIF**:
The output of `fix` or `encode`. Leaves are sorted (ascending indices, UTF-8 byte order within a scope), all bounded deterministic repairs have been applied, and the byte sequence is identical for any input that reduces to the same JSON value. `parse` is defined only on canonical RAIF; non-canonical input must go through `fix` first.

**`fix` function** (v0.4):
The pure RAIF→RAIF canonicalizing repair operation. Takes any RAIF string (RAIF-G or already-canonical RAIF), returns canonical RAIF plus an audit log of repairs applied. No model in the loop, no JSON intermediary, no schema required. Repairs are deterministic and bounded; ambiguous inputs return an error rather than guess.
_Avoid_: calling `fix` a "decoder" — it doesn't produce JSON, it produces canonical RAIF.

**`validate` function** (v0.4):
The pure RAIF check. Returns `{ ok, errors }`. Does not mutate input, does not run repairs. Answers "is this already canonical RAIF that would round-trip cleanly?" Useful as a fast pre-check before `fix` for callers that expect to receive canonical input.

**Schema-as-parity** (design principle, v0.4):
The out-of-band schema (when supplied by the caller) acts as the parity that lets the decoder resolve fundamentally ambiguous wire forms. In schema-free mode, the decoder applies only deterministic repairs whose correct interpretation is decidable from bytes alone; ambiguous cases surface as validation errors. The principle keeps RAIF a general-purpose JSON-compatible standard while letting schema-aware callers recover more.

**Deterministic repair**:
A repair the decoder applies when there is exactly one logically correct interpretation of the malformed bytes. Examples: leading-zero "number" (`02134`) → string coercion (no valid JSON number begins with `0` followed by digits); repeated-key auto-indexing (`mixed=a\nmixed=b` → `mixed[0]=a\nmixed[1]=b`) when no `mixed[N]=` already exists; nested inline-object flattening into path mode. Distinct from schema-aware repair (forbidden in schema-free mode) and from heuristic repair (rejected by ADR-0004's anti-ambiguity rule).

**Schema-typed decode** (v0.4.2, ADR-0019):
The decode mode active when the caller supplies a schema (plan §3.2 declaration syntax, e.g. `to:s`, `tags[]:s`, `note:s?`). Value types come from the schema, not value-shape inference: bare `null` under `field:s` is the string "null"; `n`/`b` must parse or surface a validation error. Eliminates the wrap-when-literal ambiguity class and recovers unwrapped pathological keys (`pathological_key_resolved`). Schema-free decode is unchanged.

**Generation profile** (v0.4.2, ADR-0019):
The emission form models are trained to emit: `encode(obj, { profile: "generation" })`. Deterministic mode precedence (table → array literal → path; nested objects always path — no cheapest-pick) and truncation-optimal ordering (single-line leaves first, multiline blocks last). Optional `<raif>`/`</raif>` framing makes truncation detectable via `decodeLenient(...).truncated`. The canonical profile (default) remains the transport/audit form.
_Avoid_: calling the generation profile "non-canonical RAIF-G syntax" — it uses the same wire grammar; only mode choice and ordering differ.

**Sparse table mode** (decode-only, v0.4):
A table-row form that uses `null` cells for missing keys, e.g. `mixed::kind,name,role\nmixed[0]=user,alice,null\nmixed[1]=group,null,admin`. Several real models naturally emit this for heterogeneous arrays; the decoder accepts it as a valid input form. The encoder does **not** produce this form — the cheapest-mode pick (ADR-0012) continues to choose between path / table / inline / array-literal. Sparse-table is decode-only canonical-input acceptance.

## Relationships

- The model emits **RAIF-G**; the interpreter parses, repairs, and canonicalizes it into **RAIF-R**.
- **Position mode** is the agent/tool-call emission shape; **path mode** is the arbitrary-JSON emission shape; **named mode** is the debugging shape.
- A **text block** carries any value whose plain encoding would corrupt the surrounding syntax; the **nonce** guarantees the block's terminators are unique on the rare multiline values where a content line would otherwise collide.
- **Table mode** is invoked automatically for homogeneous arrays; each **table header** governs the **table rows** that share its prefix.
- **Inline-object leaves** are an alternative single-leaf form picked by the **cheapest-mode pick** for heterogeneous arrays and flat nested objects, in competition with path mode and table mode.
- **Array literal** is a fourth array-emission candidate (`prefix=[\n…rows…\n]`) that shares the array prefix once across all rows; chosen by the cheapest-mode pick when its byte length is the smallest of the four candidates.
- **`fix`** is the canonical RAIF→RAIF entry point. **`decode`** composes `fix → parse → toJson`. **`encode`** is JSON → canonical RAIF. **`validate`** is the pure read-only check. These four functions are the public v0.4 API; intermediate forms (`parse`, `RaifTree`) are internal-only.
- **Sparse table mode** is one of several **non-canonical input forms** the decoder accepts via `fix`. `fix` always rewrites them into the canonical form chosen by the encoder's cheapest-mode pick.

## Flagged ambiguities

- "RAIF" alone is ambiguous — disambiguate as RAIF-G or RAIF-R unless speaking about the protocol abstractly.
- "Pathological key" (Q1) — a JSON object key containing path-significant characters (`.`, `[`, `]`, `\`) or characters that conflict with RAIF field syntax. Routed through a text block with a quoted key.
